# Legacy UI Archive

هذا المجلد يوثّق الواجهة الحالية للبرنامج بوصفها **واجهة قديمة Legacy**.

## لماذا أُرشفت

بعد مراجعة UX وIA ومسارات الاستخدام، تقرر أن:

- الواجهة الحالية ناجحة وظيفيًا
- لكنها لم تعد نقطة بداية مناسبة لإعادة التصميم
- والاستمرار فوقها سيخلط بين الإصلاحات الموضعية وبين بناء تجربة جديدة من الصفر

لذلك جرى **تجميدها مرجعيًا** بدل حذفها.

## ما الذي يعتبر Legacy الآن

- [MainWindow.xaml](../../MainWindow.xaml)
- [Views](../../Views)
- [Themes](../../Themes)
- [ViewModels](../../ViewModels)

وكذلك أي سلوك UI جديد تم تركيبه فوق هذه الطبقة يعد جزءًا من المسار القديم ما لم يُنقل لاحقًا إلى `UI_V2/`.

## ماذا يعني Legacy هنا

`Legacy` لا تعني:

- أن الكود غير صالح
- أو أن البرنامج لا يعمل
- أو أن هذا الجزء يجب حذفه فورًا

بل تعني:

- أنه لم يعد المرجع التصميمي الرسمي
- وأنه يبقى فقط للتشغيل الحالي، وللرجوع السلوكي، وللإصلاحات الحرجة عند الحاجة

## ما الذي لا نفعله بعد الآن

- لا نبني شاشة جديدة مباشرة داخل `Views/` بوصفها مستقبل المنتج
- لا نعتبر `Themes/` الحالية النظام البصري النهائي
- لا نعيد تنظيم أسماء الملاحة القديمة كحل كاف
- لا نقيس نجاح `UI_V2` بمدى قربه من هذا الأرشيف
