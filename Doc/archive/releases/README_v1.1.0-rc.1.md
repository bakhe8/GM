# GuaranteeManager v1.1.0-rc.1

هذه أول نسخة مرشحة لإطلاق `v1.1.0`. بنيت على `v1.1.0-preview.4` بعد نجاح جولة UAT معزولة على الحزمة المنشورة، وهدفها تجميد السلوك الحالي تمهيدًا للإطلاق النهائي.

> **مهم:** هذه نسخة Release Candidate. لا تضيف ميزات جديدة فوقها إلا إذا كانت إصلاحًا مانعًا ومحدود الأثر.

## حالة هذه النسخة

- رقم النسخة داخل التطبيق: `v1.1.0-rc.1`
- نوع الإصدار: **Release Candidate**
- الغرض: تأكيد جاهزية خط `v1.1.0` بعد اختصار الواجهات وتثبيت السجل الزمني كمصدر حقيقة
- اسم الحزمة: `GuaranteeManager_v1.1.0-rc.1_win-x64.zip`
- مجلد النشر: `releases/v1.1.0-rc.1`

## نطاق التجميد

- الواجهة الرسمية الحالية هي `اليوم`، `الضمانات`، `البنوك`، `التقارير`، و`الإعدادات`.
- لا توجد واجهة مستقلة للطلبات أو التنبيهات أو ملف الضمان.
- الطلبات وردود البنك والمرفقات تظهر وتدار من السجل الزمني، اللوحة الجانبية، اليوم، والتقارير.
- المستفيد ثابت: `مستشفى الملك فيصل التخصصي ومركز الأبحاث`.
- المورد هو الطرف المتغير الذي يظهر للمستخدم في الجداول، البحث، التقارير، ومخرجات Excel.

## قواعد العمل المثبتة

- طلب النقض ملغى تشغيليًا.
- الضمان المنتهي لا يقبل إلا الإفراج/الإعادة للبنك.
- التمديد والتخفيض يمكن أن يتواليا على الضمان النشط غير المنتهي.
- الإفراج والتسييل والاستبدال تنهي دورة حياة الضمان عند تأكيد البنك.
- تنفيذ إجراء نهائي يلغي الطلبات العالقة الأخرى على نفس الجذر.
- التعديل الإداري لا يغير المبلغ أو تاريخ الانتهاء أو البنك أو رقم الضمان أو النوع أو المرجع.
- السجل الزمني هو مصدر الحقيقة لدورة حياة الضمان.
