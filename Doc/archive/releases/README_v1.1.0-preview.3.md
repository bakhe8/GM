# GuaranteeManager v1.1.0-preview.3

هذه نسخة تجميد ومراجعة داخلية بعد موجة تغييرات واسعة على منطق الطلبات، توليد البيانات، والواجهات.

> **مهم:** هذه ليست نسخة إطلاق نهائي، ولا تحل محل baseline المنشور `v1.0.0`.
> الغرض منها ترتيب الشجرة بعد العمل الكبير وتحضيرها لجولة مراجعة تقود لاحقًا إلى `rc.1`.

## حالة هذه النسخة

- رقم النسخة داخل التطبيق: `v1.1.0-preview.3`
- نوع الإصدار: **Preview**
- الغرض: تثبيت baseline جديدة بعد تغييرات workflow/UI والتخلص من مسار النقض
- اسم الحزمة المتوقع: `GuaranteeManager_v1.1.0-preview.3_win-x64.zip`
- مجلد النشر المتوقع: `releases/v1.1.0-preview.3`

## أبرز ما تغيّر مقارنة بـ `preview.2`

- تثبيت منطق أن الإفراج والتسييل ينهيان دورة حياة الضمان دون إنشاء إصدار جديد.
- تثبيت أن الاستبدال ينشئ ضمانًا بديلًا ويربطه بالسلسلة القديمة دون فتح إصدار إضافي عليها.
- إلغاء طلب النقض كمسار تشغيلي كاملًا:
  - لا إنشاء.
  - لا تنفيذ.
  - لا ظهور كإجراء.
  - لا توليد بيانات.
  - قراءة البيانات القديمة فقط كـ "مسار قديم ملغى".
- إضافة بوابة اختبار على بيانات التوليد لضمان أن السيناريوهات المولدة لا تكسر قواعد workflow المثبتة.
- تحسين تمثيل أرقام الإصدارات المرتبطة بالطلبات في القوائم والتقارير.
- استمرار تنظيف سطوح `Requests`, `Guarantees`, `Dashboard`, `Banks`, `Reports`, و`Settings` حول المسارات الرسمية بدل تكرار الوظائف.
