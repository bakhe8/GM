# Repository Stabilization Checklist

**Updated:** 2026-05-03

## الهدف

تحويل الشجرة إلى baseline نظيفة بعد اختصار الواجهات، تنظيف الأرشيف، وترتيب مرشح الإطلاق الحالي.

## لا نفعل الآن

- لا نضيف ميزة جديدة قبل اكتمال التنظيف.
- لا نضيف تقارير Excel مولدة إلى git.
- لا نعيد واجهات مستقلة للطلبات أو التنبيهات أو ملف الضمان.
- لا نعيد طلب النقض بأي اسم تشغيلي.

## تحقق قبل الدمج أو الوسم

- `git status --short --branch`
- `git diff --check`
- `dotnet test .\GuaranteeManager.Tests\GuaranteeManager.Tests.csproj --no-restore`
- `dotnet build .\my_work.sln -c Debug --no-restore`

## smoke يدوي مختصر

1. افتح `اليوم`.
2. افتح `الضمانات`.
3. اختر ضمانًا وتأكد أن المورد هو الطرف الظاهر.
4. راجع السجل الزمني والمرفقات.
5. أنشئ طلبًا مؤهلًا أو سجل ردًا على طلب معلق.
6. افتح `التقارير` وتأكد من عمل تقرير ضمان أو محفظة.

## معيار RC

ننتقل من `v1.1.0-rc.3` إلى الإصدار النهائي فقط بعد:

- اختبارات نظيفة.
- بناء نظيف.
- تشغيل يدوي بلا أخطاء مانعة.
- مستودع بلا artifacts أو وثائق انتقالية خارج `Doc/archive/`.
- جولة UAT واقعية على الحزمة المنشورة.
