# Repository Structure

تصف هذه الوثيقة ترتيب المجلدات النشط بعد تنظيف 2026-05-03. الهدف أن يكون مكان كل ملف مفهوما من وظيفته، وأن تبقى المخرجات المؤقتة خارج التوثيق والجذر.

## الجذر

- `App.xaml`, `App.xaml.cs`, `GuaranteeManager.csproj`, `my_work.sln`: ملفات تشغيل وبناء التطبيق.
- `README.md`, `AI_HANDOFF.md`: بوابة المشروع وسياق الاستمرار.
- لا توضع في الجذر تقارير Excel أو لقطات شاشة أو مخرجات فحص أو ملفات تدقيق مؤقتة.

## الكود

- `Presentation/`: الواجهات الرسمية، النوافذ الفرعية، الثيمات، وسطح العمل.
- `Models/`: نماذج المجال المشتركة.
- `Services/`: خدمات التطبيق مقسمة وظيفيا:
  - `Attachments/`: حفظ وترحيل المرفقات.
  - `Backup/`: النسخ الاحتياطي والحزم المحمولة.
  - `Core/`: أخطاء وتشغيل وتسجيل عام.
  - `Database/`: تهيئة SQLite، المخطط، وطبقة الوصول للبيانات.
  - `Guarantees/`: خدمات مجال الضمان غير المصنفة ضمن التقارير أو قاعدة البيانات.
  - `Reports/`: Excel، تقارير المحفظة، وسجل الضمان.
  - `Repositories/`: مستودعات واستعلامات الضمانات والطلبات.
  - `Seeding/`: توليد بيانات اختبار/تطوير من واجهة الإعدادات.
  - `Shell/`: خدمات النافذة الرئيسية، الرسائل، الحالة، والتشخيص.
  - `Workflow/`: إنشاء الطلبات، دورة الحياة، ردود البنك، والتنفيذ.
- `Utils/`: أدوات مساعدة صغيرة. لا يضاف لها منطق مجال كبير؛ عند تضخم الأداة تنقل إلى مجلد خدمة مناسب.

## الاختبارات والأدوات

- `GuaranteeManager.Tests/`: الاختبارات الآلية.
- `Development/Diagnostics/`: أدوات تطوير خاصة لا تعمل إلا في Debug.
- `scripts/`: سكربتات النشر وفحص الواجهة.
- `scratch/`: كل مخرجات التجارب المؤقتة، بما فيها `UIAcceptance`. هذا المجلد غير متتبع.

## الأصول والتوثيق

- `Assets/`: أصول تشغيلية يحتاجها التطبيق وقت العرض أو النشر، مثل الخطوط والشعارات ورمز الريال.
- `Doc/`: التوثيق النشط.
- `Doc/design/`: الهوية البصرية، الألوان، الخطوط، وقواعد النوافذ الفرعية.
- `Doc/guides/`: أدلة التشغيل والمنطق والبنية.
- `Doc/audits/`: تقارير التدقيق والمطالبات المرتبطة بها.
- `Doc/archive/`: وثائق قديمة أو مواد مرجعية لا تدخل في دورة العمل اليومية.
- `releases/`: مخرجات النشر المحلية. يفضل إبقاء آخر إصدار فقط داخل المشروع ونقل الإصدارات القديمة لأرشيف خارجي عند التضخم.

## قواعد الإضافة

- إذا كان الملف مخرجا مؤقتا، مكانه `scratch/`.
- إذا كان الملف توثيقا نشطا، يجب ربطه من `Doc/README.md`.
- إذا كان الملف مرجعيا أو تاريخيا فقط، مكانه `Doc/archive/`.
- إذا كان الكود يخدم workflow أو تقارير أو قاعدة بيانات، لا يوضع في `Utils/`.
- عند نقل ملفات C# داخل `Services/` يفضل إبقاء namespace العام كما هو إلا إذا كان refactor مقصودا ومصحوبا باختبارات.
