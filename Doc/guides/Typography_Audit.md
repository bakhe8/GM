# Typography Audit

تاريخ المراجعة: 2026-05-02

## الخلاصة

البرنامج لا يعاني من فوضى في عائلة الخط الأساسية؛ معظم الواجهات والحوارات تستخدم عائلة واحدة تقريباً: `Segoe UI, Tahoma`.

كانت المشكلة الأوضح ليست اسم الخط، بل:

- كثرة أحجام الخطوط المتقاربة.
- وجود مقاسات كسرية كثيرة مثل `10.3`, `10.4`, `11.2`, `11.3`, `12.2`.
- عدم وجود سلم Typography معلن يحدد متى نستخدم كل مقاس.
- تكرار ضبط الخط في نوافذ وحوارات كثيرة بدلاً من الرجوع إلى مصدر واحد.

## ما تم تثبيته

- تم إنشاء `UiTypography.DefaultFontFamily` كمصدر برمجي واحد لعائلة الخط.
- تم تثبيت سلم أحجام موحد في `UiTypography`:

```text
Tiny=9, Small=10, Secondary=11, Body=12, CardTitle=13,
SectionTitle=16, Title=18, Kpi=32, HeroNumber=34
```

- تم تحويل المقاسات الكسرية والمتقاربة في الواجهة إلى هذا السلم.
- تم تحديث عائلة الخط الافتراضية إلى:

```text
Segoe UI Variable Text, Segoe UI, Tahoma
```

هذا يحافظ على خط Windows الحديث عند توفره، ويعود إلى `Segoe UI` ثم `Tahoma` عند الحاجة. هذا مناسب للنص العربي والإنجليزي والأرقام داخل واجهة Windows.

- تم تفعيل عرض نص أوضح في النافذة الرئيسية:

```text
UseLayoutRounding=True
SnapsToDevicePixels=True
TextFormattingMode=Display
TextRenderingMode=ClearType
```

## جرد أحجام الخط الحالية

قبل التنظيف كان الجرد يجد هذه الأحجام داخل ملفات الواجهة:

```text
9, 9.5, 10, 10.3, 10.4, 10.5, 10.8,
11, 11.2, 11.3, 11.5,
12, 12.2, 12.5,
13, 14, 16, 17, 18, 20, 32, 34
```

بعد التنفيذ أصبح الجرد المباشر لـ `FontSize` في ملفات الواجهة يحتوي على هذا السلم فقط:

```text
9, 10, 11, 12, 13, 16, 18, 32, 34
```

هذا هو السلم المعتمد للواجهة. تبقى بعض الأحجام المحسوبة برمجياً للأرقام الطويلة فقط حتى لا تنكسر داخل الكروت.

## السلم المقترح

السلم المعتمد:

| الدور | المقاس |
|---|---:|
| نص صغير جداً / بادجات | `9` |
| ملاحظات دقيقة | `10` |
| نص ثانوي / وصف | `11` |
| نص واجهة عادي | `12` |
| عناوين داخل الكروت | `13` |
| عناوين اللوح الجانبي | `16` أو `18` |
| أرقام كبيرة في اللوح الجانبي | `34` |
| أرقام KPI | `32` |

## الأرقام

الأرقام يجب أن تبقى واضحة وقابلة للمقارنة، خصوصاً في:

- مبالغ الضمان.
- أعداد الكروت.
- تواريخ الجداول.
- أرقام الضمان والعقود.

لا أغيّرها الآن إلى خط منفصل، لأن خلط خط عربي وخط أرقام قد يجعل الواجهة أقل طبيعية. الأفضل أولاً تثبيت العائلة الحالية، ثم مراجعة محاذاة الأرقام داخل الجداول والكروت عند الحاجة.

## الخطوة التالية

بعد تثبيت السلم، المرحلة الأفضل هي:

1. نقل أنماط النص الأساسية إلى `Typography.xaml`.
2. توحيد Helpers المكررة مثل `BuildDetailValue` و`BuildMutedText`.
3. مراجعة الجداول وحدها للتأكد أن الأرقام والتواريخ والمبالغ تظهر بمحاذاة ثابتة.
4. إضافة فحص بسيط يمنع إدخال `FontSize` خارج السلم إلا إذا كان محسوباً للأرقام الطويلة.
