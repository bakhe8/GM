# الحالة الحالية — GuaranteeManager

**تاريخ التحديث:** 2026-05-01

## الخلاصة

- النسخة الحالية: `v1.1.0-rc.1`.
- الفرع النشط: `feature/v1.1-operational-polish`.
- الواجهة الرسمية الحالية هي ما تحت `Presentation/`.
- أزيلت الأرشيفات القديمة من الشجرة حتى لا تبقى مرجعًا يوميًا مضللًا.
- لا توجد واجهة مستقلة للطلبات أو التنبيهات أو ملف الضمان.
- السجل الزمني داخل الضمان هو مصدر الحقيقة لدورة الحياة والطلبات وردود البنك والمرفقات.

## المساحات الحالية

- `اليوم`: الأولويات والمتابعات اليومية.
- `الضمانات`: المحفظة، اللوحة الجانبية، السجل الزمني، المرفقات، وتسجيل/إنشاء الطلبات.
- `البنوك`: قراءة المحفظة حسب البنك.
- `التقارير`: التصديرات والتحليلات التشغيلية.
- `الإعدادات`: النسخ الاحتياطي، الاسترجاع، المسارات، وبيانات التطوير.

## قواعد العمل المثبتة

- المستخدم هو المستفيد، وكل الضمانات تخص `مستشفى الملك فيصل التخصصي ومركز الأبحاث`.
- المورد هو الطرف المتغير الذي يظهر للمستخدم ويقود البحث والتقارير.
- طلب النقض ملغى تشغيليًا.
- الضمان المنتهي لا يقبل إلا الإفراج/الإعادة للبنك.
- التمديد والتخفيض يمكن أن يتواليا على الضمان النشط غير المنتهي.
- الإفراج والتسييل والاستبدال تنهي دورة حياة الضمان عند تأكيد البنك.
- تنفيذ إجراء نهائي يلغي الطلبات العالقة الأخرى على نفس الجذر.
- التعديل الإداري لا يغير الحقول المؤثرة في دورة الحياة مثل المبلغ أو تاريخ الانتهاء أو البنك أو رقم الضمان.
