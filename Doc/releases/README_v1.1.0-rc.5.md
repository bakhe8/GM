# GuaranteeManager v1.1.0-rc.5

هذه خامس نسخة مرشحة لإطلاق `v1.1.0`. جاءت بعد مراجعة بوابة الإصدار لإغلاق آخر مخاطر التشغيل قبل الطيار الداخلي: التقارير الثقيلة، تعافي عمليات الملفات، اختبارات النسخ الاحتياطي، وحراس البيانات الكبيرة.

> **مهم:** هذه نسخة Release Candidate مخصصة للطيار الداخلي المحدود. لا تضف عليها ميزات جديدة إلا إذا كانت إصلاحًا مانعًا ومحدود الأثر.

## حالة هذه النسخة

- رقم النسخة داخل التطبيق: `v1.1.0-rc.5`
- نوع الإصدار: **Release Candidate**
- الغرض: تثبيت خط `v1.1.0` بعد إغلاق ملاحظات بوابة الجاهزية النهائية
- اسم الحزمة: `GuaranteeManager_v1.1.0-rc.5_win-x64.zip`
- مجلد النشر: `releases/v1.1.0-rc.5`

## ما تغيّر بعد rc.4

- أصبحت إجراءات التقارير تعمل في الخلفية مع منع تشغيل تقرير ثان أثناء التنفيذ، وإظهار حالة `قيد الإنشاء` في قائمة التقارير والتفاصيل.
- أضيف طابور دائم لعمليات الملفات المؤجلة حتى تُستكمل ترقية المرفقات ومستندات رد البنك أو تنظيف الملفات عند التهيئة التالية.
- أصبحت تهيئة قاعدة البيانات تعالج طابور عمليات الملفات قبل فحص ملفات `_staging` القديمة.
- أضيفت اختبارات لتعافي المرفقات، مستندات رد البنك، والتنظيف المؤجل.
- أضيفت اختبارات إضافية لمسارات النسخ الاحتياطي: ملف وجهة مقفل، وحزمة محمولة ناقصة لا تستبدل قاعدة البيانات الحالية.
- أضيف اختبار SQL مباشر للـ `LIMIT/OFFSET`، وحارس واجهة بـ `5,000` ضمان يثبت أن مساحة الضمانات تطلب الصفحة الظاهرة فقط.
- عُدلت قواعد `.gitignore` حتى لا تحجب مجلدات المصدر `Services/Attachments` و`Services/Workflow`.

## نطاق التجميد

- الواجهة الرسمية الحالية هي `اليوم`، `الضمانات`، `البنوك`، `التقارير`، و`الإعدادات`.
- لا توجد واجهة مستقلة للطلبات أو التنبيهات أو ملف الضمان.
- الطلبات وردود البنك والمرفقات تظهر وتدار من السجل الزمني، اللوحة الجانبية، اليوم، والتقارير.
- المستفيد ثابت: `مستشفى الملك فيصل التخصصي ومركز الأبحاث`.
- المورد هو الطرف المتغير الذي يظهر للمستخدم في الجداول، البحث، التقارير، ومخرجات Excel.

## قواعد العمل المثبتة

- طلب النقض ملغى تشغيليًا.
- الضمان المنتهي لا يقبل إلا الإفراج/الإعادة للبنك.
- التمديد والتخفيض يمكن أن يتواليا على الضمان النشط غير المنتهي.
- الإفراج والتسييل والاستبدال تنهي دورة حياة الضمان عند تأكيد البنك.
- تنفيذ إجراء نهائي يلغي الطلبات العالقة الأخرى على نفس الجذر.
- التعديل الإداري لا يغير المبلغ أو تاريخ الانتهاء أو تقويم التاريخ أو البنك أو رقم الضمان أو النوع أو المرجع أو الحالة التشغيلية.
- السجل الزمني هو مصدر الحقيقة لدورة حياة الضمان.
