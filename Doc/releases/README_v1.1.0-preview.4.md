# GuaranteeManager v1.1.0-preview.4

هذه نسخة تجميد ومراجعة داخلية بعد دفعة اختصار الواجهات، تثبيت السجل الزمني كمصدر حقيقة تشغيلي، وتنظيف بنية الملفات الكبيرة.

> **مهم:** هذه ليست نسخة إطلاق نهائي، ولا تحل محل baseline المنشور `v1.0.0`.
> الغرض منها حفظ نقطة مستقرة بعد اختصار الواجهات مع بقاء منطق العمل محفوظًا في الخلفية.

## حالة هذه النسخة

- رقم النسخة داخل التطبيق: `v1.1.0-preview.4`
- نوع الإصدار: **Preview**
- الغرض: تثبيت baseline جديدة بعد تبسيط واجهات العمل وتنظيف بنية المشروع
- اسم الحزمة المتوقع: `GuaranteeManager_v1.1.0-preview.4_win-x64.zip`
- مجلد النشر المتوقع: `releases/v1.1.0-preview.4`

## أبرز ما تغيّر مقارنة بـ `preview.3`

- تثبيت السجل الزمني في واجهة الضمانات كمصدر تشغيل ومعلومة لدورة حياة الضمان.
- نقل مرفقات الأحداث وخطابات الطلبات وردود البنك إلى أفعال مباشرة داخل السجل الزمني.
- تبسيط واجهة الطلبات حتى لا تكون مركز عمل مستقل، مع بقاء منطق الطلبات والردود في الخلفية.
- إظهار كل أزرار إجراءات الضمان من لوحة الضمان نفسها، بما فيها تسجيل رد البنك، التسييل، التحقق، التخفيض، التمديد، الاستبدال، والإفراج.
- تثبيت قواعد دورة الحياة النهائية:
  - الضمان المنتهي لا يقبل إلا الإفراج/الإعادة للبنك.
  - الإفراج والتسييل والاستبدال تنهي دورة الحياة بعد تأكيد البنك.
  - تنفيذ إجراء نهائي يلغي الطلبات العالقة الأخرى على نفس الجذر.
- تصحيح معنى أطراف الضمان: المستفيد ثابت، والمورد هو الطرف المتغير المهم للبحث والعرض والتقارير.
- تحسين مخرجات Excel بإخفاء المستفيد الثابت وإضافة تقرير سلامة السجل التاريخي.
- تنظيف الأرشيفات القديمة ومسارات الواجهة المتقاعدة حتى لا تبقى مرجعًا يوميًا مضللًا.
- تقسيم الملفات الكبيرة:
  - `DataSeedingService`
  - `GuaranteeHistoryDocumentService`
  - حوارات shell
  - `ShellViewModel`
  - نماذج shell
  - اختبارات workflow الكبيرة
