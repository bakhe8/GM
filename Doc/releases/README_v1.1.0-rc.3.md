# GuaranteeManager v1.1.0-rc.3

هذه ثالث نسخة مرشحة لإطلاق `v1.1.0`. جاءت بعد `rc.2` كدفعة إصلاحات محدودة مبنية على تقارير التدقيق الخارجية، وهدفها تثبيت الإصدار قبل جولة UAT النهائية.

> **مهم:** هذه نسخة Release Candidate. لا تضيف ميزات جديدة فوقها إلا إذا كانت إصلاحًا مانعًا ومحدود الأثر.

## حالة هذه النسخة

- رقم النسخة داخل التطبيق: `v1.1.0-rc.3`
- نوع الإصدار: **Release Candidate**
- الغرض: تثبيت خط `v1.1.0` بعد إصلاحات التدقيق النهائية
- اسم الحزمة: `GuaranteeManager_v1.1.0-rc.3_win-x64.zip`
- مجلد النشر: `releases/v1.1.0-rc.3`

## ما تغيّر بعد rc.2

- أصبحت خدمة توليد البيانات واختباراتها داخلة في بناء `Release`، مع إبقاء أزرار التوليد التطويرية مخفية عن نسخة التشغيل.
- أصبحت اختبارات `Debug` و`Release` تغطي العدد نفسه من الاختبارات.
- تم تحسين واجهة الضمانات بحيث تعتمد على قاعدة البيانات في العد، الجمع، التصفية، والتصفح بدل تحميل كامل المحفظة ثم قصها داخل الذاكرة.
- تم تقليل استدعاءات قاعدة البيانات المتكررة في صفحة الضمانات بجلب طلبات الجذور المعروضة دفعة واحدة.
- تم تغليف تسجيل رد البنك وإلحاق مستند الرد بمعاملات صريحة.
- أضيفت اختبارات لمسارات فشل النسخ الاحتياطي والاسترجاع.
- أزيل الارتفاع الثابت من صف ملاحظات تقرير Excel للضمان المفرد حتى يتكيف مع المحتوى.

## نطاق التجميد

- الواجهة الرسمية الحالية هي `اليوم`، `الضمانات`، `البنوك`، `التقارير`، و`الإعدادات`.
- لا توجد واجهة مستقلة للطلبات أو التنبيهات أو ملف الضمان.
- الطلبات وردود البنك والمرفقات تظهر وتدار من السجل الزمني، اللوحة الجانبية، اليوم، والتقارير.
- المستفيد ثابت: `مستشفى الملك فيصل التخصصي ومركز الأبحاث`.
- المورد هو الطرف المتغير الذي يظهر للمستخدم في الجداول، البحث، التقارير، ومخرجات Excel.

## قواعد العمل المثبتة

- طلب النقض ملغى تشغيليًا.
- الضمان المنتهي لا يقبل إلا الإفراج/الإعادة للبنك.
- التمديد والتخفيض يمكن أن يتواليا على الضمان النشط غير المنتهي.
- الإفراج والتسييل والاستبدال تنهي دورة حياة الضمان عند تأكيد البنك.
- تنفيذ إجراء نهائي يلغي الطلبات العالقة الأخرى على نفس الجذر.
- التعديل الإداري لا يغير المبلغ أو تاريخ الانتهاء أو البنك أو رقم الضمان أو النوع أو المرجع.
- السجل الزمني هو مصدر الحقيقة لدورة حياة الضمان.
